# Visual Type Guide

Choose one type automatically unless the user names a type.

| Type | Chinese Name | Use When | Structure |
| --- | --- | --- | --- |
| `graphic-card` | 图文卡片 | One concept, definition, or quick memory point | Title, one-line explanation, 3 points, memory line, central metaphor |
| `knowledge-map` | 知识地图 | System, roadmap, capability map, classification | Center topic, 4-6 branches, short labels |
| `three-column` | 三栏结构 | Naturally has 3 dimensions, methods, modules, or categories | Title, core view, 3 parallel cards |
| `scene-story` | 场景故事 | Teacher, student, parent, workplace, or real task scenario | Role, task, AI help, human judgment |
| `analogy` | 生活类比 | Abstract AI concept or technical term | Concept, life analogy, mapping, 3 reminders |
| `process-flow` | 流程图 | Steps, method, workflow, input-to-output process | Start, 4-8 steps, end, core reminder |
| `right-wrong` | 错误与正确对比 | Misconception, bad vs good, vague vs clear | Left wrong, right correct, bottom conclusion |
| `intent-explainer` | 意图解释 | Boundary, value, "what it is / is not" | What, not what, why important, when to use |

## Selection Hints

- "什么是 X" -> `graphic-card`; if abstract, use `analogy`.
- "X 的体系 / 路线 / 能力" -> `knowledge-map`.
- "三种 / 三类 / 三步" -> `three-column`.
- "如何做 / 步骤 / 流程" -> `process-flow`.
- "误区 / 错误 / 不要这样" -> `right-wrong`.
- "老师 / 学生 / 家长 / 职场怎么用" -> `scene-story`.
- "边界 / 为什么重要 / 什么时候用" -> `intent-explainer`.

## Type Patterns

### `graphic-card`

Describe a clean knowledge card with a central visual metaphor and 3 surrounding point cards. Best for definitions and quick memory.

### `knowledge-map`

Describe a central topic node with 4-6 branch cards, simple icons, thin connector lines, and short labels. Avoid tangled lines.

### `three-column`

Describe a top title and three equal columns. Each column has one icon, one short label, and 2-3 concise points.

### `scene-story`

Describe a real teaching, study, family, or work scene. AI appears as a lightweight interface, assistant panel, or glowing card. Keep human judgment visible.

### `analogy`

Describe the life analogy visually and show how it maps to the AI concept with 3 small explanation cards. Keep it friendly but not childish.

### `process-flow`

Describe a horizontal or vertical flow with 4-8 rounded step cards connected by arrows. Each step has a short label and simple icon.

### `right-wrong`

Describe a balanced left-right comparison. Left side is vague or wrong, right side is clear or correct, with a short conclusion at the bottom.

### `intent-explainer`

Describe four cards or quadrants: "是什么 / 不是什么 / 为什么重要 / 什么时候用". Use product-doc clarity, not academic-paper density.
